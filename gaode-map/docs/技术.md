# 技术方法清单

本文档用于梳理 `map_analyse/gaode-map` 当前主分析链路中**实际使用**的技术方法，供论文撰写时引用。这里强调的是“已经在系统中落地并参与结果生成的方法”，而不是早期方案文档中提过但当前未实际运行的方法。

## 1. 研究范围界定方法

### 1.1 网络等时圈法
- 方法名称：网络等时圈（Isochrone）
- 作用：以给定中心点为起点，按照步行、骑行或驾车等出行方式，生成限定时长内可到达的空间范围，作为后续 POI、H3 网格和路网分析的统一研究边界。
- 当前实现口径：系统优先调用 `Valhalla` 等时圈服务生成真实路网可达面；当服务不可用时，退化为基于速度假设的几何缓冲近似。
- 代码依据：`modules/isochrone/adapter.py`、`modules/isochrone/core.py`

### 1.2 坐标系统一转换方法
- 方法名称：双坐标系转换与边界契约
- 作用：保证前端地图展示与后端空间分析之间的一致性。
- 当前实现口径：前端与对外接口统一使用 `GCJ02`；内部空间分析与几何运算统一使用 `WGS84`；仅在输入/输出边界进行双向转换。
- 代码依据：`docs/architect.md`、`modules/h3/core.py`

## 2. 空间单元构建方法

### 2.1 H3 六边形网格剖分法
- 方法名称：H3 六边形网格化
- 作用：将研究范围离散为统一尺度的六边形分析单元，避免行政边界或不规则地块带来的统计口径不一致问题。
- 当前实现口径：系统将研究区面域转换为 H3 单元索引，并进一步输出为可渲染的 GeoJSON 网格面。
- 代码依据：`modules/h3/core.py`

### 2.2 面-网格叠置筛选法
- 方法名称：空间叠置筛选
- 作用：确定边界格网是否纳入研究范围。
- 当前实现口径：支持 `inside` 与 `intersects` 两类保留规则，并可设置最小重叠比例，处理边界网格纳入问题。
- 代码依据：`modules/h3/core.py`

### 2.3 邻域扩展法
- 方法名称：邻域扩展（k-ring / grid-disk）
- 作用：补足边界附近可能因中心点落入规则而遗漏的 H3 网格单元。
- 当前实现口径：在 `intersects` 模式下，先基于外包矩形生成候选格，再用 H3 邻域扩展与几何相交规则共同筛选。
- 代码依据：`modules/h3/core.py`

## 3. POI 空间统计方法

### 3.1 POI 点落格聚合法
- 方法名称：POI 空间落格聚合
- 作用：将 POI 点位按照坐标归属映射到 H3 单元，形成单元级功能统计结果。
- 当前实现口径：POI 坐标先统一到分析坐标系，再通过 H3 单元索引归类到对应格网。
- 代码依据：`modules/h3/analysis.py` 中 `aggregate_pois_to_h3`

### 3.2 网格密度计算法
- 方法名称：POI 密度计算
- 作用：衡量单位面积内 POI 的集聚强度。
- 计算方式：`density = poi_count / area_km2`
- 指标单位：`POI/km²`
- 代码依据：`modules/h3/analysis.py` 中 `compute_cell_metrics`、`docs/网格分析公式对照表.md`

### 3.3 香农熵法
- 方法名称：香农熵（Shannon Entropy）
- 作用：衡量单个 H3 网格内部不同功能类别的混合程度。
- 计算方式：`H = -Σ(p_i ln p_i)`，其中 `p_i` 为某类别在该单元中的占比。
- 解释含义：熵值越高，说明功能越混合；熵值越低，说明功能越单一。
- 代码依据：`modules/h3/analysis.py` 中 `_shannon_entropy`、`docs/网格分析公式对照表.md`

### 3.4 邻域均值统计法
- 方法名称：邻域均值统计
- 作用：刻画单元周边空间环境特征，用于比较单元与邻近单元之间的差异。
- 当前实现口径：以 H3 邻域圈层为基础，计算邻域平均密度与邻域平均熵。
- 代码依据：`modules/h3/analysis.py` 中 `compute_neighbor_metrics`

## 4. 空间自相关与热点识别方法

### 4.1 全局 Moran's I
- 方法名称：全局莫兰指数（Global Moran's I）
- 作用：判断研究区整体功能密度是否具有显著空间聚集性。
- 当前实现口径：系统提供全局 Moran's I 指标作为总体空间自相关判断依据。
- 代码依据：`modules/h3/analysis.py` 中 `compute_global_moran_i`、`docs/网格分析公式对照表.md`

### 4.2 局部 Moran's I（LISA）
- 方法名称：局部空间自相关（LISA）
- 作用：识别局部高-高、高-低、低-高、低-低空间关系，用于判断局部簇团或离群现象。
- 当前实现口径：系统在 H3 局部空间结构分析中输出 `lisa_i` 等字段。
- 代码依据：`docs/网格分析公式对照表.md`

### 4.3 Getis-Ord Gi* 热点分析
- 方法名称：Gi* 热点分析
- 作用：识别局部热点与冷点的显著性强弱。
- 当前实现口径：系统输出 `gi_star_z_score`，用于判定热点和冷点等级。
- 代码依据：`docs/网格分析公式对照表.md`

### 4.4 ArcGIS 结果透传法（结构识别）
- 方法名称：ArcGIS 局部结构结果透传
- 作用：统一使用 ArcGIS 返回的局部空间统计字段，避免双引擎口径不一致。
- 当前实现口径：后端接收并透传 `gi_star_z_score`、`lisa_i`、`lisa_z_score` 等字段，不再在本仓库内做“Gi* 优先 + LISA 回退”二次分段。
- 代码依据：`modules/h3/analysis.py`、`docs/网格分析公式对照表.md`

## 5. 路网结构分析方法

### 5.1 基于 OSM 的路网提取方法
- 方法名称：OSM 路网提取
- 作用：获取研究区内道路网络，为句法分析建立原始线网基础。
- 当前实现口径：系统通过本地/私有 `Overpass` 服务抓取研究区内道路数据，并可按出行方式或道路等级进行筛选。
- 代码依据：`modules/road/core.py`、`README.md`

### 5.2 空间句法分析法
- 方法名称：空间句法分析（Space Syntax）
- 作用：从道路网络结构角度评价通达性、选择性与整体组织特征。
- 当前实现口径：当前主引擎为 `depthmapXcli`，支持 `segment` 与 `axial` 两种图模型。
- 代码依据：`modules/road/core.py`、`modules/road/schemas.py`

### 5.3 路网裁切与拓扑重建方法
- 方法名称：线网裁切与拓扑重建
- 作用：将抓取到的上下文路网裁切到研究范围内，并重建节点连接关系，保证句法指标计算与研究区边界一致。
- 当前实现口径：系统先对研究范围上下文路网进行构边，再将线段裁切到输出面内，并依据节点邻接关系重建拓扑。
- 代码依据：`modules/road/core.py`

### 5.4 路网中心性指标分析
- 方法名称：路网中心性与可读性分析
- 作用：评价道路网络在通达、选择、控制、层级组织等方面的结构特征。
- 当前实际输出指标包括：
  - `Connectivity`
  - `Choice`
  - `Integration`
  - `Control`
  - `Mean Depth`
  - `Intelligibility`
- 代码依据：`modules/road/schemas.py`、`modules/road/core.py`

### 5.5 拓扑回退控制值方法
- 方法名称：控制值拓扑回退
- 作用：在标准 depthmap 输出列缺失或近似常量时，避免控制值指标失效。
- 当前实现口径：系统基于节点邻接关系构建拓扑代理控制值，作为渲染和分析的补充结果。
- 代码依据：`modules/road/core.py`

## 6. 前端派生分析方法

### 6.1 四象限分型法
- 方法名称：高低密-高低混合四象限分型
- 作用：用于识别不同类型的功能结构单元，如高密高混合、高密低混合、低密高混合、低密低混合。
- 当前实现口径：以前端派生统计为基础，按密度与熵的分位阈值完成分类。
- 代码依据：`docs/网格分析公式对照表.md`

### 6.2 LQ 区位商法
- 方法名称：区位商（Location Quotient, LQ）
- 作用：衡量某一网格内某类功能相对于整体研究区是否具有比较优势。
- 当前实现口径：以单元内部类别占比与全局类别占比的比值作为 LQ。
- 代码依据：`docs/网格分析公式对照表.md`

### 6.3 缺口评分法
- 方法名称：功能缺口评分
- 作用：比较需求代理和供给代理之间的相对差异，用于识别潜在供给不足区域。
- 当前实现口径：以前端派生方式，将需求代理与目标业态供给强度做百分位比较，得到 `gap_score`。
- 代码依据：`docs/网格分析公式对照表.md`

## 8. 不建议写成“当前已实际使用”的方法

以下方法曾出现在早期规划或技术草案中，但从当前主链路实现看，不应直接写成“已经实际用于当前结果生成”的方法：

- 核密度估计（KDE）
- 地理加权回归（GWR）
- 栅格分区统计（如 `rasterstats`）
- 基于地块的主分析链路

这些内容可以写成“预留扩展方向”或“早期设计考虑”，但不宜在论文方法部分当作当前系统的既有分析步骤。
