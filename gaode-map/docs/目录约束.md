# 目录约束（v1）

## 1. 根目录约束
- `core/`：配置、异常、通用模型；跨业务域共享的空间/坐标工具统一收敛到这里。
- `router/`：路由聚合、依赖注入、HTTP边界。
- `modules/`：业务域实现（禁止散落 *_service 新目录）。
- `store/`：数据库模型与仓储。
- `runtime/`：运行时生成数据目录（如图表导出临时文件）。
- `templates/`、`static/`：兼容层；`/analysis` 不再依赖模板入口。
- `frontend/`：Vite + Vue3 + TS 前端工程。
- `tests/`：按 `api/domain/integration/e2e` 分层。

## 2. modules 二阶约束
- 业务域目录仅允许：`admin`、`charting`、`isochrone`、`poi`、`population`、`nightlight`、`h3`、`road`、`export`、`providers`、`map_manage`。
- Provider 统一放到 `modules/providers/*`（当前为 `amap`）。
- 已下线 legacy 目录：`grid_h3`、`road_syntax`、`export_bundle`、`gaode_service`、`isochrone_service`、`generated_charts`。
- `core/spatial.py` 是 polygon payload 解析、几何修复、GCJ02/WGS84 转换、scope id 生成的唯一共享入口；禁止在业务模块内继续复制近似实现。

## 3. 维护落点约束
- `router/*` 只允许保留参数接收、依赖注入、权限/鉴权、响应编排；采样、几何裁剪、坐标转换、缓存键生成等逻辑必须下沉到 `modules/*` 或 `core/*`。
- 空间分析域新增逻辑优先落到现有子职责文件：`facade`、`dataset`、`render`、`aggregate`、`bridge`、`cache`、`geometry`、`overpass`；不要继续把新代码堆回单一 `service.py` 或 `core.py`。
- `store/*` 只保留仓储职责；历史覆盖、摘要聚合、导出编排等业务规则应停留在对应 domain facade。
- `modules/road/core.py` 只做路网分析编排；Depthmap 命令、指标统计、GeoJSON/WebGL 序列化、进度状态分别落到子模块。
- `modules/h3/analysis.py` 只做 H3 facade 编排；类别规则、统计、ArcGIS 桥接封装必须独立维护。
- `modules/history/service.py` 承接 history 业务规则；`router/domains/history.py` 不允许绕过 service 直接拼装 repo 结果。
- 仅在确实需要替换实现或测试替身时才引入 `Protocol`；默认保持函数式和模块化组织，避免为了抽象而抽象。

## 4. 热点文件阈值
- `router/domains/isochrone.py` 不应超过 120 行。
- `modules/population/facade.py` 不应超过 500 行。
- `modules/export/builder.py` 不应超过 750 行。
- `modules/road/core.py` 不应超过 900 行。
- `modules/h3/analysis.py` 不应超过 320 行。
- `store/history_repo.py` 不应超过 220 行。
- `router/domains/road.py` 不应超过 120 行。
- `main.py` 不应超过 180 行。
- 超过阈值时，优先拆分职责，不接受继续向热点文件追加无关逻辑。

## 5. 测试分层约束
- `tests/api/`：HTTP接口级测试。
- `tests/domain/`：纯业务核心测试。
- `tests/integration/`：跨模块协同测试。
- `tests/e2e/`：端到端行为测试。

## 6. 仓库卫生约束
- 禁止提交：`__pycache__/`、`*.pyc`、`.pytest_cache/`、`.playwright-cli/`、`*.bak`。
- 禁止引用 legacy import：`modules.grid_h3`、`modules.road_syntax`、`modules.export_bundle`、`modules.gaode_service`。
- 禁止恢复旧 analysis 链路：`templates/analysis.html`、`static/js/analysis/**`。
- 使用 `../scripts/check_repo_hygiene.sh` 做结构、热点文件阈值与垃圾文件检查。
